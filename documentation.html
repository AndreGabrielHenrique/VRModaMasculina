VR Moda Masculina — Migration Notes

This document summarizes the migration to Vite + React and the current project layout.

What changed

  • Project migrated to Vite + React with componentized source under src/.
  • Styles converted to indented Sass (.sass) and organized in src/styles/.
  • Sass now uses @use with a shared _variables.sass file.
  • CSS Modules are used for per-component scoping (example: _product-card.module.sass).
  • Playwright E2E tests included under e2e/. Run with npm run test:e2e after installing browsers (npx playwright install).
  • Original static site files moved to legacy/ for reference (including legacy/index.original.html).

How to run

npm install
npm run dev  # dev server on port 5173
npm run build
npm run preview

# E2E tests:
npx playwright install
npm run test:e2e

Notas importantes

  • Se ocorrerem erros no dev server (esbuild), execute `npm install` e verifique a versão do Node. Mensagens de erro no terminal costumam indicar dependências faltantes.
  • Imagens (em src/components/Imagens/ e Imagens/) agora são importadas como módulos nos componentes — isso garante otimização pelo Vite e suporte a tree-shaking.
  • Comentários explicativos em português foram adicionados a todos os arquivos fonte — descrevem imports, estado, hooks, funções e lógica principal. Revise e edite conforme sua preferência antes do commit final.
  • Arquivo documentation.html e readme.md foram atualizados com status atual. Verifique antes de commitar.
    • src/ — código React, entry em src/main.jsx e src/App.jsx.
    • src/components/ — Header, Carousel, Products, ProductCard, Cart, CheckoutForm, Footer, ScrollButton, icons.
    • src/styles/ — partials SASS e entry style.sass.
    • e2e/ — testes Playwright (homepage, cart interactions, visual snapshots).
    • legacy/ — versão estática original (HTML/CSS/JS) preservada.

    Próximos passos recomendados

    1. Revisar localmente com git status e git diff.
    2. Rodar npm run dev e, se for o caso, npm run test:e2e.
    3. Corrigir qualquer divergência visual/funcional e só então commitar em um único commit, se essa for a sua política.
